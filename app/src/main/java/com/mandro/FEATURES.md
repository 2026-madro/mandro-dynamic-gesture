## 1. 제품 목표

### 핵심 목표
기존에 연구자가 옆에서 직접 안내하던 EMG 개인화 학습 과정을 **앱이 완전히 대체**한다.

유저 혼자서:
1. 센서를 연결하고
2. 앱의 가이드에 따라 6가지 동작을 수행하고
3. 서버에서 개인 모델을 학습받고
4. Teensy 펌웨어에 업로드하고
5. 실시간으로 제스처 분류 결과를 확인한다

### 제품 단계

| 단계 | 목표 | 기간 |
|------|------|------|
| v1 | 데스크탑 앱 기능을 Android로 완전 이식 | 1개월 |
| v2 | 서버 데이터 누적 + 베이스 모델 개선 | 추후 |
| v3 | Fine-tuning 기반 즉시 사용 (온보딩 최소화) | 추후 |

## 2. 기능 명세서

### F01 — BLE 연결

| 항목 | 내용 |
|------|------|
| 설명 | EMG 센서 디바이스를 BLE로 스캔 및 연결 |
| 우선순위 | P0 (필수) |
| 입력 | 유저가 스캔 버튼 탭 |
| 출력 | 연결된 디바이스 표시, 신호 수신 시작 |
| 예외 | 연결 실패 시 재시도 안내, 블루투스 꺼져있으면 권한 요청 |

**세부 요구사항:**
- BLE 스캔 → 디바이스 목록 표시
- 디바이스 선택 후 자동 연결
- 연결 상태 상단 상시 표시 (연결됨 / 끊김 / 연결 중)
- 백그라운드에서 연결 유지
- 연결 끊김 시 자동 재연결 시도 (3회)

---

### F02 — 실시간 파형 시각화

| 항목 | 내용 |
|------|------|
| 설명 | BLE로 수신한 8채널 EMG 신호를 실시간 파형으로 표시 |
| 우선순위 | P0 (필수) |
| 입력 | BLE 데이터 스트림 (1200 Hz, 8채널) |
| 출력 | 8채널 실시간 스크롤 파형 |

**세부 요구사항:**
- 8채널 파형 동시 표시 (채널별 색상 구분)
- 스크롤 속도 조절 가능
- 신호 강도 (amplitude) 자동 스케일링
- 특정 채널 on/off 토글
- 파형 일시정지 / 재개

---

### F03 — 유저 관리

| 항목 | 내용 |
|------|------|
| 설명 | 피험자(유저) 등록 및 선택 |
| 우선순위 | P0 (필수) |
| 입력 | 이름, ID 입력 |
| 출력 | 유저 프로필 생성, 이후 모든 데이터에 연결 |

**세부 요구사항:**
- 유저 생성 (이름 + 자동 ID 부여)
- 기존 유저 선택 (모델 있으면 바로 사용 가능)
- 로컬 유저 목록 저장 (Room DB)
- 서버와 유저 ID 동기화

---

### F04 — 학습 가이드 (온보딩)

| 항목 | 내용 |
|------|------|
| 설명 | 6가지 동작을 유저에게 시각적으로 안내 |
| 우선순위 | P0 (필수) |
| 입력 | 유저 탭 |
| 출력 | 동작별 설명 + 애니메이션 표시 |

**6가지 동작 안내 내용:**

| 동작 | 설명 | 안내 포인트 |
|------|------|------------|
| Rest | 손을 자연스럽게 펴서 쉬기 | 긴장 풀기, 기준 자세 |
| Flexion | 손목을 아래로 구부리기 | 천천히, 끝까지 |
| Extension | 손목을 위로 젖히기 | 천천히, 끝까지 |
| Close (Fist) | 주먹 꽉 쥐기 | 손가락 전부 굽히기 |
| Supination | 손바닥이 위를 향하게 돌리기 | 팔꿈치 고정 |
| Pronation | 손바닥이 아래를 향하게 돌리기 | 팔꿈치 고정 |

**세부 요구사항:**
- 동작당 일러스트 or 애니메이션
- 텍스트 설명 + 주의사항
- "이해했어요" 확인 후 다음으로
- 언제든지 다시 볼 수 있는 가이드 메뉴

---

### F05 — 데이터 수집

| 항목 | 내용 |
|------|------|
| 설명 | 동작별 EMG 데이터를 녹화하고 레이블링 |
| 우선순위 | P0 (필수) |
| 입력 | 유저가 동작 수행, 앱이 타이밍 안내 |
| 출력 | 레이블된 EMG CSV 파일 (로컬 저장) |

**세부 요구사항:**
- 동작당 10회 반복 수집 (기본값, 설정 가능)
- 수집 중 카운트다운 + 진행 상황 표시
- 수집 중 실시간 파형 표시
- 수집 완료된 동작 체크 표시
- 불량 데이터 재수집 가능 (개별 take 재녹화)
- 로컬 임시 저장 (앱 종료 후 이어서 가능)

**채널 품질 모니터링:**
- 평소 녹화 중에는 채널 상태를 화면에 표시하지 않음 (깔끔한 UI 유지)
- **채널 연결 끊김**(하드웨어 접촉 불량) 감지 시에만 즉시 경고 배너 표시
    - 예: "CH3 연결이 불안정해요. 암밴드를 다시 착용해보세요"
- 끊김이 발생한 take는 내부적으로 플래그 처리 (`hasChannelDropout: Boolean`)

### F05-B — 녹화 품질 리포트 (신규)

| 항목 | 내용 |
|------|------|
| 설명 | 6개 동작 전체 녹화 완료 후, 채널 끊김이 있었던 동작에 한해 재녹화를 안내 |
| 우선순위 | P0 (필수) |
| 입력 | 6개 동작 녹화 완료 |
| 출력 | 재녹화 필요 동작 목록 또는 "학습 시작하기" 진입 |

**세부 요구사항:**
- 6개 동작을 리스트로 표시, 각각 ✓(정상) 또는 ⚠(끊김 발생) 표시
- ⚠ 표시된 동작만 "다시 녹화하기" 버튼 제공 (타겟팅 재녹화, 전체 재녹화 아님)
- 끊김 없는 동작이라면 리포트 화면 생략하고 바로 학습 진행 가능
- 모든 동작이 ✓ 상태가 되어야 "학습 시작하기" 버튼 활성화
- 데스크탑 앱의 채널별 연결 표시 + 사후 진단 메시지를 모바일 친화적으로 단순화한 버전

---

### F06 — 서버 학습 요청

| 항목 | 내용 |
|------|------|
| 설명 | 수집된 데이터를 FastAPI 서버로 업로드 후 학습 요청 |
| 우선순위 | P0 (필수) |
| 입력 | 수집 완료된 EMG 데이터 |
| 출력 | 학습된 `.keras` 모델 + `scaler.pkl` |

**세부 요구사항:**
- 데이터 업로드 진행률 표시
- 서버 학습 중 대기 UI (예상 시간 표시)
- 학습 완료 후 모델 파일 자동 다운로드
- 실패 시 재시도
- 오프라인 상태면 "나중에 학습" 예약 가능

---

### F07 — 펌웨어 업로드 (USB)

| 항목 | 내용 |
|------|------|
| 설명 | 학습된 모델을 USB로 Teensy 펌웨어에 플래시 |
| 우선순위 | P0 (필수) |
| 입력 | 학습된 모델 파일, USB 연결된 Teensy |
| 출력 | 펌웨어 업로드 완료 |

**세부 요구사항:**
- USB Host API 사용 (OTG 지원 기기)
- USB 연결 감지 자동 안내
- 업로드 진행률 표시
- 업로드 완료 후 BLE 재연결 안내
- 업로드 실패 시 에러 메시지 + 재시도

---

### F08 — 실시간 분류

| 항목 | 내용 |
|------|------|
| 설명 | 개인화 모델로 실시간 제스처 분류 결과 표시 |
| 우선순위 | P0 (필수) |
| 입력 | BLE EMG 스트림 + 로컬 모델 |
| 출력 | 현재 제스처 분류 결과 + 신뢰도 |

**세부 요구사항:**
- 분류 결과 실시간 표시 (제스처명 + 신뢰도 %)
- 8채널 파형과 분류 결과 동시 표시
- 분류 히스토리 (최근 N개)
- 신뢰도 낮을 때 경고 표시

## 4. 전체 순서도

### 4-1. 최초 사용 (신규 유저)

```
앱 실행
  │
  ▼
유저 생성 (이름 입력)
  │
  ▼
BLE 연결 (디바이스 스캔 → 선택 → 연결)
  │
  ▼
학습 가이드 열람 (6가지 동작 설명)
  │
  ▼
데이터 수집 루프
  ├── 동작 안내 표시 (동작명 + 일러스트)
  ├── 카운트다운 (3, 2, 1)
  ├── 녹화 (3초 유지)
  ├── 완료 / 재수집 선택
  └── 6동작 × 10회 완료까지 반복
  │
  ▼
서버 업로드 + 학습 요청
  ├── 데이터 압축 업로드
  ├── 서버 학습 대기 (진행률 폴링)
  └── 모델 파일 다운로드 (model + scaler)
  │
  ▼
USB 펌웨어 업로드
  ├── USB 연결 안내
  ├── Teensy 감지 확인
  ├── 모델 플래시
  └── 완료 확인
  │
  ▼
BLE 재연결
  │
  ▼
실시간 분류 화면
```

### 4-2. 재방문 (기존 유저, 모델 있음)

```
앱 실행
  │
  ▼
유저 선택 (기존 유저 목록)
  │
  ▼
BLE 연결
  │
  ▼
실시간 분류 화면 (바로 진입)
```

```

  │
  ▼
데이터 수집 (추가 배치)
  │
  ▼
서버 학습 요청 (기존 데이터 + 신규 데이터)
  │
  ▼

  │
  ▼

### 5-1. 전체 구성

│                                             │
│  ┌──────────┐   ┌──────────┐   ┌─────────┐ │

│  │ Manager  │   │ Manager  │   │ Client  │ │
│  └────┬─────┘   └────┬─────┘   └────┬────┘ │
│       │              │              │       │

│  │           Domain Layer                │  │

         │ HTTPS                   │ USB OTG
         ▼                         ▼

```

### 5-2. Android 앱 레이어 구조 (Clean Architecture)

```
Presentation Layer  →  Jetpack Compose UI + ViewModel
Domain Layer        →  UseCase (학습, 수집, 분류 비즈니스 로직)
Data Layer          →  Repository (BLE, HTTP, DB, File)
```

### 5-3. FastAPI 서버 구조

```
POST /users                    # 유저 생성
POST /sessions/{user_id}/data  # 데이터 업로드
POST /sessions/{user_id}/train # 학습 요청
GET  /sessions/{user_id}/model # 모델 다운로드
GET  /sessions/{user_id}/status # 학습 진행 상태 폴링
```

**서버 학습 흐름:**
```
데이터 수신
  │
  ▼
기존 train_causal_concat.py 실행 (subprocess or import)
  │
  ▼
model_*.keras + scaler_*.pkl 생성
  │
  ▼
S3 or 로컬 저장
  │
  ▼
앱에 완료 응답 + 다운로드 URL
```

### 5-4. 신호 처리 흐름 (Android 내)

```
BLE 패킷 수신 (8ch raw)
  │
  ▼
링 버퍼 (128 samples)
  │
  ▼
밴드패스 필터 (Butterworth 4차, 35–300 Hz)
  │
  ▼
절댓값 + Envelope (이동평균)
  │
  ▼
피처 추출 (Kotlin)
  ├── Classic: MAV, MaxAV, STD, RMS, WL, SSC + FFT 5종 → 88차원
  └── TSD: 공분산 + 에너지 (sub-windowing) → 44차원
  │
  ▼
StandardScaler 정규화 (scaler.pkl의 mean/std 적용)
  │
  ▼
TFLite 추론 (132 → 64 → 64 → 6)
  │
  ▼
Softmax → 제스처 분류 결과 + 신뢰도
```

## 7. 개인정보 및 데이터 동의

### 7-1. 준거법

**개인정보 보호법 (대한민국)** 기준으로 설계한다.

EMG 신호는 생체정보(민감정보)에 해당할 수 있으므로 일반 개인정보보다 강화된 동의 절차를 적용한다.

---

### 7-2. 수집 항목 및 목적

| 항목 | 수집 목적 | 보유 기간 |
|------|-----------|-----------|
| 유저 식별자 (익명 UUID) | 개인 모델 연결 | 서비스 탈퇴 시까지 |
| EMG 신호 데이터 (8채널) | 개인 모델 학습 | 서비스 탈퇴 시까지 |
| EMG 신호 데이터 (선택 동의 시) | 베이스 모델 성능 개선 연구 | 동의 철회 시까지 |

> 이름, 이메일 등 직접 식별 가능한 정보는 수집하지 않는다.  
> 유저 식별자는 앱이 자동 생성하는 익명 UUID를 사용한다.

---

### 7-3. 동의 구조

동의는 **필수 동의**와 **선택 동의** 두 가지로 분리한다.

```
유저 생성 화면 (S03)
  │
  ├── [필수] 개인정보 처리방침 동의
  │         - 익명 ID 생성 및 개인 모델 학습에 EMG 데이터 사용
  │         - 미동의 시 앱 사용 불가
  │
  └── [선택] 연구 목적 데이터 제공 동의
            - 수집된 EMG 데이터를 모델 성능 개선 연구에 활용
            - 미동의 시에도 앱 모든 기능 정상 사용 가능
            - 언제든지 설정에서 철회 가능
```

---

### 7-4. UI 요구사항

**유저 생성 화면 (S03):**
- 필수 동의: 체크박스 + "전문 보기" 링크 (미동의 시 다음 버튼 비활성화)
- 선택 동의: 별도 체크박스, 기본값 **미체크**
- 각 항목 옆에 왜 필요한지 한 줄 설명 포함
- "전문 보기" 탭 시 인앱 웹뷰로 전체 개인정보 처리방침 표시

**설정 화면 (S11):**
- 선택 동의 상태 언제든지 변경 가능
- 동의 철회 시 서버에 저장된 연구용 데이터 삭제 요청 가능
- 전체 데이터 삭제 요청 기능 (회원 탈퇴)

---

### 7-5. 서버 처리 원칙

- **선택 동의한 유저의 데이터만** 베이스 모델 학습에 사용
- 서버에 전송되는 모든 데이터는 익명 UUID로만 식별
- 데이터 전송 시 HTTPS 필수
- 동의 철회 / 탈퇴 API 구현 필수:

```
DELETE /users/{user_id}/research-data   # 연구용 데이터만 삭제 (선택 동의 철회)
DELETE /users/{user_id}                 # 전체 삭제 (탈퇴)
```

---

### 7-6. 개인정보 처리방침 필수 고지 항목

개인정보 보호법 제30조 기준, 아래 항목을 처리방침에 반드시 포함한다:

1. 수집하는 개인정보 항목 (EMG 신호, 익명 UUID)
2. 수집 및 이용 목적
3. 보유 및 이용 기간
4. 제3자 제공 여부 (현재: 없음)
5. 동의 철회 방법 및 데이터 삭제 절차
6. 개인정보 처리 담당자 연락처
