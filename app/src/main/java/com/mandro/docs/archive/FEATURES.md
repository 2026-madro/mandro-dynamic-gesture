## 1. 제품 목표

### 핵심 목표
기존에 연구자가 옆에서 직접 안내하던 EMG 개인화 학습 과정을 **앱이 완전히 대체**한다.

유저 혼자서:
1. 센서를 연결하고
2. 앱의 가이드에 따라 6가지 동작을 수행하고
3. 서버에서 개인 모델을 학습받고
4. Teensy 펌웨어에 업로드하고
5. 실시간으로 제스처 분류 결과를 확인한다

### 제품 단계

| 단계 | 목표 | 기간 |
|------|------|------|
| v1 | 데스크탑 앱 기능을 Android로 완전 이식 | 1개월 |
| v2 | 서버 데이터 누적 + 베이스 모델 개선 | 추후 |
| v3 | Fine-tuning 기반 즉시 사용 (온보딩 최소화) | 추후 |

### F01 — BLE 연결 ✅ UI / 🔌 실제 연동 필요

| 항목 | 내용 |
|------|------|
| 설명 | EMG 센서 디바이스를 BLE로 스캔 및 연결 |
| 우선순위 | P0 (필수) |
| 입력 | 유저가 스캔 버튼 탭 |
| 출력 | 연결된 디바이스 표시, 신호 수신 시작 |
| 예외 | 연결 실패 시 재시도 안내, 블루투스 꺼져있으면 권한 요청 |

**세부 요구사항:**
- BLE 스캔 → 디바이스 목록 표시
- 디바이스 선택 후 자동 연결
- 연결 상태 상단 상시 표시 (연결됨 / 끊김 / 연결 중)
- 백그라운드에서 연결 유지
- 연결 끊김 시 BLE 스캔 화면으로 이동

> **변경:** 연결 끊김 시 자동 재연결(3회) 대신 BLE 화면으로 튕기는 방식으로 결정

### F02 — 실시간 파형 시각화 ✅ UI / 🔌 실제 연동 필요

|------|------|
| 설명 | BLE로 수신한 8채널 EMG 신호를 실시간 파형으로 표시 |
| 우선순위 | P0 (필수) |
| 입력 | BLE 데이터 스트림 (1200 Hz, 8채널, int16) |

- 신호가 없는 채널 확인 안내 문구 표시

> **제거:** 파형 일시정지/재개 기능 — 불필요하다고 판단해 제거  
> **제거:** 스크롤 속도 조절 기능  
> **미결:** 신호 강도 자동 스케일링 (현재 int16 고정 스케일)

### F04 — 학습 가이드 (온보딩) ✅ UI 완료

| 항목 | 내용 |
|------|------|
| 설명 | 6가지 동작을 유저에게 시각적으로 안내 |
| 우선순위 | P0 (필수) |
| 입력 | 유저 탭 |
| 출력 | 동작별 설명 + 애니메이션 표시 |

**6가지 동작 안내 내용:**

| 동작 | 한국어 설명 | 안내 포인트 |
|------|------------|------------|
| Rest | 손을 자연스럽게 펴서 쉬기 | 긴장 풀기, 기준 자세 |
| Flexion | 손목을 아래로 구부리기 | 천천히, 끝까지 |
| Extension | 손목을 위로 젖히기 | 천천히, 끝까지 |
| Close | 주먹 꽉 쥐기 | 손가락 전부 굽히기 |
| Supination | 손바닥이 위를 향하게 돌리기 | 팔꿈치 고정 |
| Pronation | 손바닥이 아래를 향하게 돌리기 | 팔꿈치 고정 |

**세부 요구사항:**
- 동작당 일러스트 애니메이션 — Lottie 자리 확보, 에셋 미준비
- 텍스트 설명 + 주의사항 카드
- 슬라이드 없이 진행 바가 부드럽게 증가하는 스테퍼 방식으로 전환
- 마지막 동작에서 "동작 녹화 시작하기"로 수집 화면 진입
- 등록 탭 진입 시 바로 가이드 화면으로 연결

### F05 — 데이터 수집 ✅ UI 완료 / 🔌 실제 녹화 필요

| 항목 | 내용 |
|------|------|
| 설명 | 동작별 EMG 데이터를 랩 단위로 녹화하고 레이블링 |
| 우선순위 | P0 (필수) |
| 입력 | 유저가 동작 수행, 앱이 타이밍 안내 |
| 출력 | 레이블된 EMG 데이터 (→ 서버 전송) |

**수집 구조 (랩 기반):**
- 1랩 = 6개 동작 × 10초
- 총 10랩 수집 목표
- **최소 5랩 이상**이어야 학습 시작 가능 (5랩 완료 시 "지금 학습하기" 버튼 활성화)
- 각 동작마다 3초 카운트다운 → 10초 녹화 자동 반복

> **변경:** 초기 설계(동작당 10회 반복)에서 **랩 기반 수집**으로 변경  
> 이유: 같은 동작 연속 반복 대비 근육 피로도 분산, 진행 체감 향상

**세부 요구사항:**
- 카운트다운 3→2→1 (원형 UI, scale+fade 애니메이션)
- 녹화 중 동작 일러스트 표시 + 남은 초 카운트다운 ("동작 중... 7초")
- 랩 진행 바 상단 표시
- 이번 랩 내 동작 현황 리스트 (완료 ✓ / 현재 진행 / 대기)
- 5랩 달성 전: "최소 5랩 이상 수집해야 학습할 수 있어요" 안내
- 5랩 이상 달성 시: "지금 학습 시작하기" 버튼 노출
- DEV 빌드 전용 "다음 화면으로 →" 스킵 버튼 (`BuildConfig.DEBUG`)

**채널 품질 모니터링 — 🔌 미구현:**
- 채널 연결 끊김(하드웨어 접촉 불량) 감지 시 즉시 경고 배너
- 끊김 발생 take는 내부 플래그 처리 (`hasChannelDropout: Boolean`)

### F06 — 서버 학습 요청 ✅ UI / 🔌 실제 연동 필요

|------|------|
| 설명 | 수집된 데이터를 FastAPI 서버로 업로드 후 학습 요청 |

| 입력 | 수집 완료된 EMG 데이터 |

**세부 요구사항:**
- 각 단계별 상태: 대기(회색) → 진행 중(파란 원) → 완료(초록 체크)
- 업로드 진행률 % 표시
- 예상 시간 표시, "앱을 닫지 말고 기다려 주세요"
- 실패 시 재시도 — 🔌 미구현
- 오프라인 "나중에 학습" 예약 — 🔌 미구현

### F07 — 펌웨어 업로드 (USB) ✅ UI / 🔌 실제 연동 필요

| 항목 | 내용 |
|------|------|
| 설명 | 학습된 모델을 USB로 Teensy 펌웨어에 플래시 |
| 우선순위 | P0 (필수) |
| 입력 | 학습된 모델 파일, USB 연결된 Teensy |
| 출력 | 펌웨어 업로드 완료 |

**세부 요구사항:**
- USB 연결 감지 → 3단계 체크 순차 확인 (케이블 연결 → 암밴드 감지 → 설정 준비)
- 모든 체크 완료 시 "업데이트 시작" 버튼 활성화 + "모두 확인했어요!" 메시지
- 업데이트 중 일러스트 → 로딩 스피너 전환
- 완료 후 동작 인식 탭으로 자동 이동
- "나중에 할게요" 없음 — 업데이트 없이는 분류기 사용 불가하므로 필수

> **제거:** "나중에 할게요" 버튼 — 학습 후 펌웨어 업데이트 없이는 기기 사용 불가하므로 제거

### F08 — 실시간 분류 ✅ UI / 🔌 실제 연동 필요

| 항목 | 내용 |
|------|------|
| 설명 | 개인화 모델로 실시간 제스처 분류 결과 표시 |
| 우선순위 | P0 (필수) |
| 입력 | BLE EMG 스트림 + 로컬 모델 |
| 출력 | 현재 제스처 분류 결과 + 채널별 신호 벡터 시각화 |

**레이더 차트 시각화:**
- 8채널을 좌우 각 4개씩 대칭 배치 (±30°, ±60°, ±120°, ±150°)
- 각 채널값(0~1 정규화)만큼 중심에서 선이 뻗어나감
- 채널 끝점 연결선 없음, 끝점 표시 없음
- `awaitFrame()` + FloatArray 직접 읽기로 레이턴시 최소화
- 동심원 3개로 기준 강도 표시

**세부 요구사항:**
- 인식된 제스처명 + 한국어 설명 카드 (다크 테마)
- "인식이 잘 안 되나요? → 다시 학습하기" → 등록 탭으로 이동

> **변경:** 신뢰도(%) 표시 제거 — 의미 없다고 판단  
> **변경:** 최근 인식 기록 히스토리 제거  
> **변경:** 파형 별도 표시 없음 — 신호 탭에서 별도 확인

## 4. 전체 순서도

### 4-1. 최초 사용 (신규 유저)

```
앱 실행
  │
  ▼
유저 생성 (이름 입력 + 개인정보 동의)
  │
  ▼
BLE 연결 (디바이스 스캔 → 선택 → 연결)
  │
  ▼
[등록 탭] 학습 가이드 열람 (6가지 동작 스테퍼)
  │
  ▼
[등록 탭] 동작 녹화 (랩 기반)
  ├── 3초 카운트다운 → 10초 녹화 자동 반복
  ├── 6동작 완료 = 1랩
  ├── 5랩 달성 시 "지금 학습하기" 활성화
  └── 10랩 완료 or 유저 선택 시 학습 진행
  │
  ▼
[등록 탭] 서버 업로드 + 학습 요청 (4단계 스테퍼)
  │
  ▼
[등록 탭] USB 펌웨어 업데이트
  │
  ▼
[인식 탭] 실시간 분류 (레이더 차트)
```

### 4-2. 재방문 (기존 유저, 모델 있음)

```
앱 실행 → 유저 선택 → BLE 연결 → 인식 탭 (바로 진입)
```

### 4-3. 재학습

```
인식 탭 "다시 학습하기" → 등록 탭 (가이드부터 다시)
```

## 5. 시스템 아키텍처

### 5-1. 전체 구성

```
┌─────────────────────────────────────────────┐
│              Android 앱                      │
│                                             │
│  ┌──────────┐   ┌──────────┐   ┌─────────┐ │
│  │   BLE    │   │   USB    │   │  HTTP   │ │
│  │ Manager  │   │ Manager  │   │ Client  │ │
│  └────┬─────┘   └────┬─────┘   └────┬────┘ │
│       │              │              │       │
│  ┌────▼──────────────▼──────────────▼────┐  │
│  │           Domain Layer                │  │
│  │  EMG Signal Processor                 │  │
│  │  Feature Extractor (Kotlin)           │  │
│  │  Model Inference (TFLite)             │  │
│  └───────────────────┬───────────────────┘  │
│                      │                      │
│  ┌───────────────────▼───────────────────┐  │
│  │            Data Layer                 │  │
│  │  Room DB (users, sessions)            │  │
│  │  Local File Storage (models, CSVs)    │  │
│  └───────────────────────────────────────┘  │
└─────────────────────────────────────────────┘
         │ HTTPS                   │ USB OTG
         ▼                         ▼
┌─────────────────┐       ┌─────────────────┐
│  FastAPI Server │       │  Teensy 펌웨어  │
│                 │       │  (EMG 센서 +    │
│  /train         │       │   분류기)       │
│  /model         │       └─────────────────┘
│  /data/upload   │
└─────────────────┘
```

### 5-2. Android 앱 레이어 구조 (Clean Architecture)

```
Presentation Layer  →  Jetpack Compose UI + ViewModel
Domain Layer        →  UseCase (학습, 수집, 분류 비즈니스 로직)
Data Layer          →  Repository (BLE, HTTP, DB, File)
```

### 5-3. FastAPI 서버 구조

```
POST /users                    # 유저 생성
POST /sessions/{user_id}/data  # 데이터 업로드
POST /sessions/{user_id}/train # 학습 요청
GET  /sessions/{user_id}/model # 모델 다운로드
GET  /sessions/{user_id}/status # 학습 진행 상태 폴링
```

### 5-4. 신호 처리 흐름 (Android 내)

```
BLE 패킷 수신 (8ch raw, int16)
  │
  ▼
링 버퍼 (200 samples per channel)
  │
  ▼
밴드패스 필터 (Butterworth 4차, 35–300 Hz)
  │
  ▼
절댓값 + Envelope (이동평균)
  │
  ▼
피처 추출 (Kotlin)
  ├── Classic: MAV, MaxAV, STD, RMS, WL, SSC + FFT 5종 → 88차원
  └── TSD: 공분산 + 에너지 (sub-windowing) → 44차원
  │
  ▼
StandardScaler 정규화 (scaler.pkl의 mean/std 적용)
  │
  ▼
TFLite 추론 (132 → 64 → 64 → 6)
  │
  ▼
Softmax → 제스처 분류 결과
```

## 7. 개인정보 및 데이터 동의

### 7-1. 준거법
**개인정보 보호법 (대한민국)** 기준. EMG 신호는 생체정보(민감정보)에 해당할 수 있으므로 강화된 동의 절차 적용.

### 7-2. 수집 항목 및 목적

| 항목 | 수집 목적 | 보유 기간 |
|------|-----------|-----------|
| 유저 식별자 (익명 UUID) | 개인 모델 연결 | 서비스 탈퇴 시까지 |
| EMG 신호 데이터 (8채널) | 개인 모델 학습 | 서비스 탈퇴 시까지 |
| EMG 신호 데이터 (선택 동의 시) | 베이스 모델 성능 개선 연구 | 동의 철회 시까지 |

### 7-3. 동의 구조

유저 생성 화면(S03)에서 필수/선택 동의 분리:
- **[필수]** 개인정보 처리방침 동의 — 미동의 시 앱 사용 불가
- **[선택]** 연구 목적 데이터 제공 동의 — 미동의 시에도 모든 기능 사용 가능
